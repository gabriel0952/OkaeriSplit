## ADDED Requirements

### Requirement: 使用者可從收據圖片辨識消費品項
系統 SHALL 允許使用者拍照或從相簿選取收據/發票圖片，透過兩段式本地端 AI（ML Kit OCR + 系統 LLM）理解圖片內容，輸出結構化 JSON 並解析擷取品項名稱、數量與總金額。

#### Scenario: 拍照辨識收據
- **WHEN** 使用者在新增費用頁點擊「掃描收據」並選擇拍照
- **THEN** 系統開啟相機，拍照後進行圖片預處理，依序執行 ML Kit OCR 與系統 LLM 推論，顯示辨識中 loading 狀態（「AI 正在分析收據...」）

#### Scenario: 從相簿選取收據圖片
- **WHEN** 使用者在新增費用頁點擊「掃描收據」並選擇從相簿選取
- **THEN** 系統開啟相簿，選取後進行圖片預處理、OCR 與 LLM 推論

#### Scenario: 辨識成功
- **WHEN** LLM 成功解析 OCR 文字並輸出至少一個品項或金額
- **THEN** 系統進入辨識結果編輯頁，顯示解析出的品項列表（名稱、數量、金額）與總金額

#### Scenario: 辨識失敗或無法解析
- **WHEN** OCR 無法擷取文字、LLM 推論失敗（超時）或 JSON 解析後無法擷取任何有效品項與金額
- **THEN** 系統顯示錯誤提示，提供重新掃描選項

---

### Requirement: 裝置不支援時降級處理
系統 SHALL 在不支援系統 LLM 的裝置上隱藏掃描入口或顯示不支援提示，不允許使用者進入辨識流程。

#### Scenario: 不支援裝置隱藏入口
- **WHEN** 使用者的裝置不支援 Apple Intelligence（iOS < 18.1 或非 A17 Pro 以上晶片）或 Android AICore
- **THEN** `AddExpenseScreen` 不顯示「掃描收據」按鈕

#### Scenario: 進入頁面時再次確認可用性
- **WHEN** 使用者以某種方式進入 `ReceiptScanResultScreen`，但裝置不支援 LLM
- **THEN** 系統顯示「此裝置不支援 AI 辨識功能」訊息，不執行辨識

---

### Requirement: 支援中文、英文、日文收據辨識（自動，無需選擇）
系統 SHALL 支援辨識繁體中文、英文、日文收據，由 OCR 擷取原文後 LLM 自動理解，使用者無需手動選擇語言。

#### Scenario: 辨識繁體中文收據
- **WHEN** 使用者拍攝台灣超商或餐廳收據（繁體中文）
- **THEN** ML Kit OCR 擷取中文文字，LLM 解析後品項名稱以原文輸出

#### Scenario: 辨識日文收據
- **WHEN** 使用者拍攝日本收據（日文）
- **THEN** ML Kit OCR 擷取日文文字，LLM 解析後品項名稱以原文輸出

#### Scenario: 辨識英文收據
- **WHEN** 使用者拍攝英文收據
- **THEN** ML Kit OCR 擷取英文文字，LLM 解析後品項名稱以原文輸出

---

### Requirement: 收據圖片解析為結構化資料
系統 SHALL 將 LLM 輸出的 JSON 字串透過 `LlmReceiptParser` 轉換為結構化品項資料。

#### Scenario: 解析標準格式收據
- **WHEN** LLM 輸出包含明確的品項名稱與金額 JSON
- **THEN** 解析器擷取各品項的名稱、數量、金額，以及總金額

#### Scenario: 解析含數量的品項
- **WHEN** LLM 輸出中品項包含 quantity 欄位（> 1）
- **THEN** 解析器擷取品項名稱、數量與小計金額

#### Scenario: 僅辨識到總金額
- **WHEN** LLM 輸出無法解析出個別品項，但包含 total 金額
- **THEN** 結果僅包含總金額，品項列表為空，使用者可手動新增品項

#### Scenario: LLM 輸出格式不正確
- **WHEN** LLM 輸出帶有 markdown code fence 或前綴文字
- **THEN** 解析器自動去除 code fence、定位 JSON 邊界後解析；若仍無法解析則回傳空 items

---

### Requirement: 辨識結果可預覽與編輯
系統 SHALL 提供辨識結果編輯介面，使用者可在匯入費用表單前修正品項資料。

#### Scenario: 編輯品項名稱與金額
- **WHEN** 使用者在辨識結果頁點擊某品項
- **THEN** 該品項的名稱與金額欄位變為可編輯狀態，修改後即時更新總金額

#### Scenario: 刪除品項
- **WHEN** 使用者在辨識結果頁滑動刪除或點擊刪除某品項
- **THEN** 該品項從列表移除，總金額即時更新

#### Scenario: 新增品項
- **WHEN** 使用者在辨識結果頁點擊「新增品項」
- **THEN** 列表底部新增一筆空白品項，使用者可輸入名稱與金額

#### Scenario: 修改總金額
- **WHEN** 使用者手動修改總金額
- **THEN** 系統保留使用者輸入的總金額（不自動以品項金額加總覆蓋）

---

### Requirement: 辨識結果可匯入費用表單
系統 SHALL 允許使用者將辨識結果一鍵匯入新增費用表單，自動填入對應欄位。

#### Scenario: 匯入辨識結果至費用表單
- **WHEN** 使用者在辨識結果頁點擊「匯入」
- **THEN** 系統返回新增費用頁，自動填入：總金額、說明文字（「收據掃描」）、切換為項目拆分模式並填入各品項、收據圖片加入附件列表

#### Scenario: 費用表單已有資料時匯入
- **WHEN** 使用者在已填入部分資料的費用表單中執行掃描並匯入
- **THEN** 系統以辨識結果覆蓋金額與品項，但保留使用者已選擇的付款人與分攤成員設定
