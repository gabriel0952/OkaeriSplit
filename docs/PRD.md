# OkaeriSplit PRD（Product Requirements Document）

## 1. 產品概述

- **產品名稱**：OkaeriSplit（おかえり Split）
- **定位**：簡潔直覺的多人分帳 APP，讓記帳分帳像回家一樣輕鬆
- **目標用戶**：台灣市場優先（中文介面，預設 TWD），架構預留 i18n
- **平台**：Android（API 24+）/ iOS
- **競品參考**：Splitwise（改善其介面複雜、總覽不清晰、建帳流程繁瑣的問題）

## 2. 使用情境

| 情境 | 說明 | 範例 |
|------|------|------|
| 室友/合租 | 長期固定群組，持續記錄日常開銷 | 房租、水電、日用品 |
| 旅行分帳 | 短期活動，多幣別消費 | 機票、住宿、餐飲、門票 |
| 聚餐/活動 | 單次活動費用分攤 | 餐廳、KTV、團購 |

## 3. 核心功能

### 3.1 群組管理

- 建立群組（名稱、類型、預設幣別）
- 邀請成員：邀請碼加入 + 搜尋用戶帳號直接邀請
- 訪客成員：無需對方事先註冊即可加入群組
- 群組成員管理（加入/退出/刪除群組）
- 群組封存與重新開啟
- 產生網頁分享連結

### 3.2 記帳

- **基本資訊**：金額、付款人、日期、描述、備註
- **消費分類**：餐飲、交通、住宿、娛樂、日用品、其他（可自訂）
- **附件**：收據/照片拍照或相簿選取後上傳
- **多幣別**：支援不同貨幣記帳

### 3.3 分帳方式

- **均分**：所有參與者平均分攤
- **自訂比例**：按比例分配（如 2:1:1）
- **指定金額**：手動指定每人金額
- **項目拆分**：單筆消費中個別項目指定由誰付

### 3.4 結算

- 欠款總覽：誰欠誰多少，一目了然
- **最簡化轉帳**：自動計算最少轉帳次數（債務簡化演算法）
- 手動標記已付款
- 結算歷史記錄

### 3.5 訪客成員

- 群組成員可新增「訪客」：無需對方事先註冊，由建立者代入姓名
- 訪客透過「群組碼 + 訪客碼」兩段驗證方式登入，確保只有本人能認領
- 訪客可升級為正式帳號（輸入 Email + 密碼），歷史消費與群組資料全數保留
- 訪客可主動退出群組（退出後帳號自動刪除）

### 3.6 群組封存

- 群組擁有者可封存群組：封存後停止所有消費/結算寫入
- 封存時自動移除所有訪客成員（清理未使用的訪客帳號）
- 封存群組仍可瀏覽歷史記錄，Owner 可隨時重新開啟

### 3.7 網頁分享

- 群組成員可產生分享連結（3 個月有效期）
- 透過連結可在瀏覽器查看群組消費摘要，無需安裝 App 或登入
- 支援消費明細展開、多幣別顯示

### 3.8 帳號與同步

- 註冊/登入（Email + Google Sign-In + Apple Sign-In）
- 忘記密碼：發送重設信件，透過 Deep Link 回到 App 設定新密碼
- 雲端資料即時同步（Supabase Realtime）
- 離線記帳：無網路時可新增消費（本地暫存），網路恢復後自動同步
- 多裝置支援

### 3.6 總覽與統計

- 個人總覽：我欠別人 / 別人欠我的彙總
- 群組消費統計（按分類圓餅圖、按月份趨勢）
- 消費搜尋與篩選（關鍵字、分類、付款人、日期範圍）
- 清晰的 Dashboard 設計

## 5. 設計原則

- **簡潔優先**：新增消費常用路徑 ≤ 3 步完成（金額 → 描述 → 送出），進階功能以漸進揭露收折
- **一眼看懂**：Dashboard 讓使用者立刻知道自己的帳務狀態
- **快速操作**：常用功能（記帳、查欠款）要能最快到達
- **視覺品質**：Apple 冷白系設計語言——乾淨底色、精緻字距、卡片以背景色差代替陰影

### 5.1 視覺設計方向

- **風格定位**：Apple iOS/macOS 現代簡約風
- **色彩系統**：Indigo 主色 (#4F46E5)，淺灰底色 (#F5F5F7)，白色卡片，語義色（綠=應收、紅=應付）
- **表單設計**：Card 分區（各功能群組獨立 Card），固定底部送出按鈕

### 5.2 新增消費互動原則

- **分區清晰**：描述+分類、付款人、分帳成員、更多選項各自一張 Card
- **智慧預設**：幣別沿用群組預設、日期預設今天、分攤成員預設全選、分帳方式預設均分
- **固定送出**：送出按鈕固定在底部 SafeArea，不隨表單捲動
- **Disabled 條件**：金額 ≤ 0 或描述空白時 disabled

## 6. 技術棧

> 詳見 `CLAUDE.md`

- Flutter 3.41.0 / Dart 3.x
- 狀態管理：Riverpod
- 導航：go_router
- 後端：Supabase (Auth + PostgreSQL + Realtime + Storage)
- Data class：freezed
- 錯誤處理：fpdart (Either)
- CI/CD：Xcode Cloud

## 7. 非功能需求

- **效能**：列表滑動 60fps，啟動 < 2 秒
- **安全**：Supabase 憑證透過 `--dart-define-from-file` 注入，不進版控；API 通訊 HTTPS
- **離線**：支援離線記帳（Hive 本地暫存）與自動同步；讀取降級為 Hive cache
